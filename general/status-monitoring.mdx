# Status Monitoring

EKB provides a real-time status page where you can monitor system health, track ongoing issues, and stay informed about platform availability and performance.

## Overview

The EKB status page (`status.getodin.ai`) provides:

- **Real-Time Status** - Live monitoring of all platform services
- **Incident Tracking** - Current and historical incidents
- **Service Status** - Individual service health indicators
- **Maintenance Updates** - Scheduled maintenance notifications
- **Performance Metrics** - System performance and uptime statistics

## Accessing the Status Page

### Direct Access

Visit the status page directly:

**URL:** [https://status.getodin.ai](https://status.getodin.ai)

### From Documentation

The status page can be accessed from:

- Status page link in the documentation
- Support resources
- System notifications

## Understanding the Status Page

### Service Status Indicators

The status page displays the current operational status of each service:

**Operational (Green)**

- Service is functioning normally
- No known issues affecting the service
- Full availability and performance

**Issues Detected**

- Service is experiencing problems
- Some functionality may be impacted
- Team is working on resolution

The status page clearly indicates whether each service (App, API, Database Health, Chat API) is operational or experiencing issues.

### Monitored Services

The status page monitors the following services:

**App**

- Main application service
- User interface availability
- Application functionality

**API**

- REST API endpoints
- API availability and performance
- API response times

**Database Health**

- Database connectivity
- Database performance
- Data availability

**Chat API**

- Chat service endpoints
- Real-time chat functionality
- Chat API response times

## Monitoring Live Issues

### Current Status

The status page provides real-time updates on service availability:

- **Service Status** - Current operational status of each service
- **Real-Time Updates** - Live status information
- **Service Health** - Quick view of overall platform health

### Active Incidents

When incidents occur, the status page displays:

- **Incident Information** - Details about ongoing issues
- **Affected Services** - Which services are impacted
- **Status Updates** - Real-time updates on incident resolution
- **Timeline** - Chronological updates on the incident

## Historical Incidents

### Incident History

The status page maintains a log of past incidents:

- **Past Service Disruptions** - Historical incidents and their impact
- **Incident Details** - Information about what happened during each incident
- **Resolution Information** - How incidents were resolved
- **Service Impact** - Which services were affected by each incident

### Incident Archive

Review detailed information about:

- **Incident Timeline** - When incidents occurred and their duration
- **Affected Services** - Which services were impacted
- **Resolution Status** - How and when issues were resolved
- **Lessons Learned** - Improvements made after incidents

## Scheduled Maintenance

### Upcoming Maintenance

View scheduled maintenance windows:

- **Maintenance Window** - Date and time of maintenance
- **Affected Services** - Services that will be unavailable
- **Expected Duration** - How long maintenance will take
- **Impact** - What functionality will be affected

### Maintenance Notifications

Stay informed about:

- Maintenance announcements
- Schedule changes
- Completion updates
- Post-maintenance status

### Response Times

1. **Check Status Page** - Regularly check for active incidents

## Troubleshooting

### Status Page Not Loading

**Problem:** Status page is not accessible

**Solutions:**

1. Check your internet connection
2. Try accessing from a different network
3. Clear browser cache
4. Try a different browser
5. Check if status page itself is experiencing issues

### Not Receiving Notifications

**Problem:** Not receiving status update emails

**Solutions:**

1. Check spam/junk folder
2. Verify email subscription is active
3. Check email notification preferences
4. Re-subscribe if needed
5. Verify email address is correct

### Status Shows Operational But Issues Persist

**Problem:** Status shows green but you're experiencing issues

**Solutions:**

1. Check if issue is specific to your account
2. Review recent incident updates
3. Check for regional issues
4. Contact support with specific error details
5. Check service-specific status indicators
